Routing Engine
Final Result Determination
The routing algorithm uses Q5 (needles), Q6 (insurance intent), Q7 (budget/brand), and Q10 (insurance status) as primary inputs. Q3 (weight goal) determines semaglutide vs. tirzepatide sub-routing within each result.
Q6 (Insurance) Q7 (Priority) Q5 (Needles) Q10 (Status) Result
“Yes, use insurance” Any Any Employer / Private Insurance Program
“Yes, use insurance” Any Any Medicaid/Medicare Self-Pay + note re: coverage limitations
“Yes, use insurance” Any Any No / Not sure Self-Pay (insurance unavailable)
“No, self-pay” “Most affordable” Fine / Prefer avoid Any Self-Pay Compounded (sema default)
“No, self-pay” “Most affordable” Absolutely not Any Oral Program (Glow Rx Appetite)
“No, self-pay” “Brand-name” Fine Any Branded Rx Management
“No, self-pay” “Brand-name” Prefer avoid / No Any Branded Rx (Wegovy oral highlighted)
“No, self-pay” “Best results” Fine Any Self-Pay Tirzepatide
“No, self-pay” “Best results” Prefer avoid / No Any Oral Program (Glow Rx Metabolic Reset)
“Not sure” Any Fine Any “Yes” Hybrid: Insurance + Self-Pay shown
“Not sure” Any Any No / Not sure Self-Pay (routed by Q3 + Q7)

Needle Override Rule

If Q5 = “Absolutely not — no needles,” the result ALWAYS routes to the Oral Program regardless of other answers — UNLESS they specifically selected “brand-name” in Q7, in which case show Wegovy Oral as the primary recommendation with Glow Rx products as alternatives.

Dev Notes
Implementation Requirements

HIPAA Compliance

No medical answers (Q2 contraindications) are ever stored with identifying information. The contraindication screen is anonymous — email capture only happens on the disqualification pop-up AFTER the quiz terminates, stored in a separate system from medical answers. Phase 1 and Phase 2 answers must live in separate data stores from the email captured in Phase 3.

Discount Codes by Result

Tracking & Attribution

Each result page = unique URL for attribution. UTM parameters from the ad source must carry through the full quiz into the result page URL and Shopify checkout. Quiz completion events should fire to: Meta Pixel, Google Ads, TikTok Pixel, and Zoho CRM via n8n webhook.

Mobile-First Design Requirements

Every screen must be thumb-friendly. Max 4 answer options visible without scrolling on any question. Interstitial cards should be swipeable. Sticky bottom CTA bar on mobile throughout. Progress bar at top showing completion percentage.

Abandoned Quiz Recovery

If a user reaches Q9 (email capture) but doesn’t submit, track this as an abandoned quiz event. If they previously entered any partial data through a cookie or session, fire a retargeting pixel event for “quiz abandoner” audience. No email recovery is possible since email hasn’t been captured yet — this is ad retargeting only.
